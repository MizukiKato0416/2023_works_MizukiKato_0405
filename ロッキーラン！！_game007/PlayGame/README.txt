//ゲームタイトル
ロッキーラン！！

//制作期間
2021年10月〜2021年11月　(約1ヶ月)

//開発環境
Visual Studio 2015　,　C++/DirectX9

//制作人数
5人

//ルール
コントローラーを2つ接続して遊ぶ1対1の対戦型ゲームです。
1Pは分割されている画面の左にいるロッキーくん、2Pは右にいるロキ子ちゃんを操作してゴールを目指します。
ゴールした際にはゴールまでにかかったタイムが表示され、1対1の勝敗だけでなくランクインを目指すことができます。
ゴールまでの道のりにギミックとして飛び移る床、氷の床、移動する壁、落とし穴、バネがあります。
プレイヤーはダイブをすることによってジャンプよりも遠くに飛び移ることができますが、ダイブ使用後には若干の
硬直が生まれます。

//操作方法
XInputのゲームパッドでのみ遊ぶことができます。
LStick	 			：プレイヤー移動／カーソル移動
RStick	 			：視点移動
START				：ポーズ／ポーズ解除／次に進む／決定
A					：ジャンプ
B					：ダイブ(ジャンプ中のみ)／次に進む／決定
十字ボタン			：カーソル移動
ESC	  	 			：ゲームを終了する

//パッド操作ができなかった際の動作確認用　(動くのは1Pのみ)
W/A/S/D	 			：プレイヤー移動／カーソル移動
十字ボタン	 		：視点移動／カーソル移動
ENTER				：次に進む／決定
P					：ポーズ／ポーズ解除
SPACE				：ジャンプ／ダイブ(ジャンプ中のみ)
ESC	  	 			：ゲームを終了する

担当部分：camera／control_player／fade／input／input_keyboard／input_mouse／input_pad_x／light／main／manajer／
		　meshfield／meshsphere／model／model_single／motion_player／number／play_data／ranking／renderer／scene／
		　scene2D／scene3D／score／texture／x_load

//アピールポイント
今回の制作はプログラムが苦手な人でも絶対に1部分は処理を実装できるよう、C++でのプログラムができる人たちで教え合い、
協力して取り組みました。
また、制作に取り掛かる際、メンバーそれぞれの発想を活かしたゲームを制作するため全員の意見や構想、アイデアをメンバーの
前で出してもらい、私がそれらをまとめてアレンジを加え、全員が納得のいくゲーム作りをしました。
処理の部分としてアピールしたいポイントはcontrolクラスを考案したことです。実際にソースコードを書いてもらったのは
別のメンバーですが私がどのようにして作るのか教えました。
controlクラスを作ることによって1つのクラスで複数の動きを持つUI等の作業を細分化し、分担できるため作業効率が大幅に
上がりました。これにより制作に余裕が生まれ、ゲームのクオリティアップが出来ただけでなく、チーム全員がしっかりと休息を
とって制作に臨めるようになりました。
また、落とし穴については棒倒し法というアルゴリズムを用いて生成したので、ステージを生成するたびに最低1本の道が
ランダムにできるようになっています。
